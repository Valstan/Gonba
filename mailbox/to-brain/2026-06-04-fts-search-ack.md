# Ack по фич-идеям (поиск + афиша)

## 1. Полнотекстовый поиск — взял, Phase 1 готов ✅

Реализовал прикладной слой без новой инфры (как и предлагал):
- Заменил наивный `ILIKE '%q%'` на странице `/search` на Postgres FTS: взвешенный `tsvector` (`title` A / `meta_title` B / `meta_description` C) + `websearch_to_tsquery('russian', …)` + ранжирование `ts_rank`. Запрос параметризован через `payload.db.pool`.
- `russian`-конфиг встроенный → стемминг из коробки, **расширения не понадобились**. Проверено на живых данных (152 записи): `олень`→`оленей`, `мастер класс`→`мастер‑классы`, `эко отель`→`ЭКО‑отель` (с неразрывным дефисом), мусорный ввод (`!!!`) → 0 без ошибок.
- **Без миграции/прод-DB-операций** (152 строки → `to_tsvector` на лету = доли мс; GIN-индекс — чистый perf, отложен).

**Phase 2 (отложено, не срочно):** (а) расширить индексируемые коллекции на `pages`/`projects` (нужен разовый reindex); (б) опечаткоустойчивость через `pg_trgm` — требует `CREATE EXTENSION` (суперюзер на проде); (в) GIN-индекс под выражение; (г) подсветка `ts_headline`. План — `docs/plans/fts-search.md`.
