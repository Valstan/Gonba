# Build→CI: выполнено

- **PR #134 + фикс #135** смержены, **первый CI-деплой зелёный** (run 27341348995): build в раннере (Node 20 = мажор бокса, `npm ci`) → standalone-артефакт → `releases/<sha>` → симлинк `current` → юнит идемпотентно из репо → restart → smoke-гейты (local health, CDN, контент-маркер #011) — все прошли. Cutover юнита атомарный, ручных шагов между merge и деплоем не было.
- **Prerender из живой прод-БД через SSH-туннель** отработал штатно (design-решение из предыдущего письма): маркер главной на месте, `[slug]`-контент в SSR, sitemap перегенерён сборкой. Чанки из live-HTML — все 200 (манифест цел). Визуальная гидратация браузером — отложена (Chrome-расширение оффлайн на этой машине), класс ChunkLoadError проверен курлом чанков.
- **Твой прогноз по рантайму сбылся:** standalone ужал RSS **452 → 254 МБ** (плюс ушла npm-обвязка ~50 МБ). Фон load 0.00.
- **Диск (suggest):** с подтверждением владельца удалён старый on-box `web/.next` (590 МБ) и swap ужат 8→2 ГБ → **итого 20.45 ГБ → 14 ГБ занято (72% → 49%, свободно 15 ГБ)**. KARMAN media есть куда заезжать.
- Бокс-репо оставлен (нужен таймерам vk-sync/media-cache и ручным миграциям); деплой обновляет его `git pull` + `npm install` только при изменении package-lock.json.

## Грабля для GOTCHAS (переносимая, мелкая)

**Два lockfile'а в репо — проверь, какой канонический, ДО написания CI-джоба.** У нас на дисках лежал `web/pnpm-lock.yaml` (локально все работают pnpm), но он в `.gitignore` — в репо живёт только `package-lock.json` (им собирается ci.yml). Первый CI-деплой упал на `pnpm install --frozen-lockfile`: ERR_PNPM_NO_LOCKFILE. Симптом обманчив: локально lockfile «есть». Рецепт: канонический lockfile = тот, которым живёт существующий CI; новые джобы — тем же менеджером. Fail-safe порядок шагов окупился: упало до туннеля/билда/шипа, прод не тронут.
