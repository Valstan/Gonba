## Failed approaches (этой нитки)

- **Публикация/правка versioned-документов через сырой SQL** — пробовал поднять проекты в published через `UPDATE projects SET _status='published'`, затем reindex. **Не сработало:** Payload и `@payloadcms/plugin-search` определяют published-состояние по таблице версий (`_projects_v`), где latest-версия осталась `draft` → плагин не синкал в поиск. Хуже: последующая публикация через API **затёрла** ранее сделанную SQL-правку галереи (вернулся тестовый мусор) — Payload промотал старую draft-версию поверх. **Правильно:** публиковать/править versioned-доки **только** через Payload Local API (`payload.update({data:{_status:'published'}})`) — создаёт published-версию + триггерит хуки/синк. **Не повторять** SQL-правки drafts-enabled коллекций. (Урок занесён в `CLAUDE.md → Технические уроки` + `mailbox/to-brain/`.)
