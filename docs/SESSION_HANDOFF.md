## Failed approaches (этой нитки)

- **Параллельные/перекрывающиеся деплои → ChunkLoadError (прод-outage 2026-06-05).** При деплое site-decor запустил ОДНОВРЕМЕННО ручной `gh workflow run deploy-prod.yml` И авто-деплой от merge → `.next` пересобирался во время отдачи → манифест чанков рассинхронился → клиент запросил layout-чанк, которого уже не было → «Application error» на ВСЕХ страницах. **Корень — не код декора** (nginx — чистый `proxy_pass`, parens-чанки `app/(frontend)/...` отдаются 200; код переедан без изменений и заработал). **Правило:** деплой — строго ПО ОДНОМУ. После merge ждать ЕДИНСТВЕННЫЙ авто-деплой, **не** запускать параллельно ручной dispatch. (Занесено в `CLAUDE.md → Технические уроки`.)
- **Smoke-check деплоя (HTTP 200 + контент-маркер) НЕ ловит client-side ChunkLoadError** — деплой #111 «прошёл успешно» (200 + orbit-маркер в SSR-HTML), а клиент был сломан. Вывод: после деплоя фронт-изменений **визуально проверять гидратацию в браузере** (Claude-in-Chrome), не доверять только smoke. (Класс «зелёный пайплайн ≠ корректный результат», родственно pool #011.)
