## Failed approaches (этой сессии)

- **Dry-run, который пишет.** Первая версия бэкафилл-скрипта вызывала `downloadDedupMedia` (создаёт Media) **до** проверки флага `--dry` → dry-run писал бы в Я.Облако. Исправлено: в DRY только считаем фото, скачивание/`payload.update` под `!DRY`. Урок: в data-скриптах ставь dry-гейт **перед** любым сайд-эффектом, не только перед финальной записью.
- **owner_id всегда `-groupId`.** Для личных страниц (VK user pages, напр. группа 86086407) `wall.getById` с отрицательным owner возвращает пусто. Фикс: fallback на `+userId` если `-groupId` пусто. (Slug старых импортов не различает user/community — знак приходится подбирать.)
- **jsonb LIKE без пробела.** Проверка `content::text LIKE '%"type":"upload"%'` даёт ложный негатив — Postgres рендерит jsonb с пробелом после двоеточия (`"type": "upload"`). В SQL ищи с пробелом; в JS (`JSON.stringify`) — без.
