# Plans — многоэтапные планы разработки

В этой папке лежат **планы из plan mode** Claude Code и любые другие многоэтапные дорожные карты, которые нужно протащить через несколько сессий.

---

## Зачем тут, а не в `~/.claude/plans/`

`~/.claude/plans/` — папка Claude Code outside-repo. У неё две проблемы:

1. **Не видна с другого компа** — git её не передаёт.
2. **Не видна пользователю в репо** — нельзя ткнуть в PR.

Поэтому: при создании плана через plan mode сохраняй в `docs/plans/<slug>.md` (file path можно настроить через `plansDirectory` в settings.json, но проще — писать сразу сюда).

---

## Формат

Имя файла — `<short-slug>.md` (например `media-to-yadisk.md`, `auth-refactor.md`).

Внутри — что хочется. Минимум:

- **Цель** — 1-2 предложения.
- **Этапы** — нумерованный список с дельтами по сессиям.
- **Текущий этап** — где мы сейчас в плане.
- **Связь с PENDING_FOLLOWUPS / ADR** — куда смотреть за деталями.

---

## Lifecycle

- **Создание** — при начале многоэтапной задачи. Если задача укладывается в одну сессию — план не нужен.
- **Обновление** — после завершения этапа отметить как пройденный, скорректировать остальное по результатам.
- **Закрытие** — когда все этапы пройдены: переименовать в `<slug>-DONE.md` или удалить (история останется в git log).

`docs/SESSION_HANDOFF.md` ссылается на активный план через поле «План: docs/plans/X.md». Если плана нет — handoff просто говорит «—».
