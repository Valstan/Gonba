# Заимствования из Sabantuy → Гоньба (программа оживления сайта)

**Решение владельца:** 2026-07-10 — одобрены все 4 пакета.
**Источник:** sibling-репо `../SabantuyMalmyzh` (тот же стек Next 15 + Payload 3 + PG; чтение read-only по ADR-0007). Код адаптируем, не копипастим вслепую: у Гоньбы медиа живёт на Я.Диске (ADR-0001), свои роли/access, свой дизайн (лесной зелёный/янтарь/глина).

## Ключевые принципы (из опыта Sabantuy)

- **Degraded-без-секретов:** S3/VK-app/VAPID при отсутствии кредов молча выключают фичу (endpoint 503 / кнопка скрыта), сборка и типы зелёные. Владелец добавляет секреты когда удобно.
- **Прямая загрузка presigned PUT в Yandex Object Storage** — UGC-медиа мимо слабого бокса; в БД только objectKey. Существующую Media/Я.Диск НЕ трогаем (UGC — отдельный контур).
- **ПДн-минимизация (152-ФЗ):** IP только sha256-хешем; VK-вход берёт только vkId+имя+аватар.
- **Анонимное владение:** ownerToken в localStorage → X-UGC-Owner → сервер хранит ownerHash. VK-вход — второй слой (ownerVisitor), не обязательный.
- Rate-limit in-memory, стоп-фильтр мата, жалобы с авто-скрытием по порогу.

## Этапы

### Этап 1 — Народная лента (UGC core) ⏳
- Коллекции: `Submissions` (подпись + до 20 медиа objectKey), `SubmissionComments`, `SubmissionReactions`, `ContentReports`.
- `lib/s3.ts` (presigned PUT, path-style, public-read), `POST /api/ugc/sign-upload`, `/api/ugc/{mine,delete-submission,...}`.
- Страница `/lenta`: лента постов, загрузка (файлы + камера), лайтбокс, комментарии, реакции, жалобы.
- Постмодерация в /admin (скрыть/удалить), стоп-фильтр, rate-limit, ownerToken.
- **От владельца:** создать бакет Yandex Object Storage + ключи в `/etc/gonba/gonba.env` (`S3_*`). До этого — degraded.

### Этап 2 — Фотостена + фотобатл
- `/fotostena`: masonry всех одобренных кадров (лента + опц. галереи проектов), порционный рендер, единый лайтбокс.
- Фотобатл: пары кадров, battleWins/battleShows COUNT-хуком, лидерборд месяца = фильтр по календарю (без cron), страница статистики.

### Этап 3 — Афиша + «Моя программа»
- Апгрейд витрины существующих Events: таймлайн по дням, звёздочки (localStorage), экспорт .ics / share, обратный отсчёт до ближайшего.
- Без фестивальной специфики («идёт сейчас» оставить — полезно и для деревенских событий).

### Этап 4 — Викторина-краеведение
- `QuizQuestions` (форматы: вопрос-варианты / правда-миф / по картинке; обязательный source), каталог игр в коде, `/igra/[game]`, прогресс в localStorage.

### Этап 5 — VK ID вход + web-push + PWA
- VK OAuth 2.1 PKCE, коллекция `visitors`, HMAC-cookie отдельно от payload-token; владение UGC с любого устройства. **От владельца:** VK-приложение (App ID + secret).
- Web-push (VAPID, без сезонного авто-стопа) о новых постах/событиях; PWA-манифест + SW + офлайн. **От владельца:** сгенерить VAPID разрешу сам (openssl), секрет в gonba.env.

## Известные грабли (из Sabantuy HANDOFF/GOTCHAS)
- G102 ISR force-static — нас не касается (build против прод-БД), но новые роуты проверять заголовком.
- VK-шаринг IDN-домена теряет картинку сниппета — баг ВК, обхода нет.
- G135 FK-мины — новые relationship `required:true` не заводить без осознания FK SET NULL.
