# Inbox from brain_matrica

> Сюда приходят **заявки** от meta-репо [`brain_matrica`](../../../brain_matrica/) — оркестратора всех проектов разработчика.

## Что это

Brain_matrica — отдельный приватный repo (`Valstan/brain_matrica`), где живут:
- Реестр всех проектов
- Pool переносимых идей между проектами
- Tech-radar
- Cross-project ADRs

**Brain_matrica-сессия (Claude в meta-репо)** наблюдает за всеми проектами и время от времени формулирует «заявки» — предложения попробовать что-то в этом проекте. Заявки кладутся в эту папку.

## Что **не** делать с заявками

- **НЕ выполнять автоматически.** Каждая заявка — гипотеза, не директива.
- **НЕ воспринимать как баг-репорт.** Brain_matrica не видит код этого проекта, только meta-данные.
- **НЕ переносить в `PENDING_FOLLOWUPS.md` без анализа.** Сначала оцени применимость.

## Что **делать** с заявками

Когда `/start` находит файл `inbox-from-brain/NNNN-*.md` (не README), выведи в отчёте:

```
📨 Заявка от brain_matrica: #NNNN — <title>
   Применить / Отложить / Отклонить?
```

При **рассмотрении**:
1. Прочитай заявку **целиком** — там обязательная шапка-маркер 🧠 «без полного контекста проекта».
2. Сопоставь с текущим состоянием проекта (`SESSION_HANDOFF`, `PENDING_FOLLOWUPS`, текущая нитка).
3. Реши: применить полностью / частично / отложить / отклонить.
4. **Запиши решение** в `DEVELOPMENT_LOG.md` (или эквивалент) с пометкой `brain_matrica dispatch #NNNN`.
5. **Удали файл** заявки из inbox-from-brain (или перемести в `archive/`, если хочешь сохранить историю).

## Формат каждой заявки

Шапка фиксированная: маркер 🧠 + Target + Created + Status + Контекст у brain_matrica + Предложение + Зачем + Возможные «нет». См. примеры в самой папке или [`brain_matrica/dispatch/PROTOCOL.md`](../../../brain_matrica/dispatch/PROTOCOL.md).

## Лимиты

- Brain_matrica соблюдает: **не более 3 активных заявок** на проект одновременно.
- Заявка «истекает» через 30 дней без рассмотрения — brain_matrica при следующем `/start` помечает её `superseded`.
