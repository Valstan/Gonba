# 0004. Главная страница — редизайн на «этно-модерн» (направление B)

- **Status:** Accepted
- **Date:** 2026-05-23
- **Deciders:** Valstan (владелец), Claude (Гоньба-сессия 2026-05-23)

## Context

Текущая главная — `HomeCarouselMenuClient` (orbit-карусель) + `HomeProjectGridMobile`. Шапка списком выгружает все 10 проектов, в том числе 3 проекта с английскими slug'ами (`eco-hotel-booking`, `about-project`, `vyatskiy-sbor`), которым так и не дали русских заголовков.

Пользователь принёс в сессию архив handoff'а из Claude Design — три направления визуального языка (A Журнал / B Этно-модерн / C Сторителлинг) и production-ready HTML главной для направления B. К каждому направлению приложен `audit.jsx`, где разобраны 6 ключевых проблем текущего сайта:

1. Слаги вместо названий в навигации.
2. Плоский список 10+ ссылок, нет иерархии.
3. Нет позиционирования — главная сразу падает в перечень разделов.
4. Фото не герой — orbit-карусель ставит UI выше натуры.
5. Нет «что делать» — турист хочет сценарий, мы даём структуру.
6. Мобильный — второй сорт, хотя у туризма основной трафик мобильный.

Проблемы реальные, узнаваемые. Пользователь попросил «принять решение, что взять оттуда, и обмозговать, не пострадает ли проектность» (10 проектов могут потеряться при переходе на 4-категориальную раскладку).

Архив (после распаковки) лежит в [`docs/design/handoff-2026-05-23/`](../design/handoff-2026-05-23/INDEX.md).

## Decision

Берём **направление B (Этно-модерн)** как основу для редизайна главной. Архитектуру коллекций Payload не трогаем — все 10 проектов остаются `Projects`, добавляем поле `group` (stay/do/see/shop) для иерархии.

**Сопутствующие решения:**

- **Orbit-карусель** (`HomeCarouselMenuClient`) переезжает с главной на отдельную страницу `/orbit` как «всё одним взглядом». Не удаляется — пользователю нравится визуально.
- **`/projects`** (`EditableProjectsGrid` с inline-редактированием) остаётся как плоский каталог-каскадник. Эта страница уже умеет admin-режим — терять её незачем.
- **Развёртывание** — поэтапно, 4 PR'а (см. [`docs/plans/etno-modern-redesign.md`](../plans/etno-modern-redesign.md)). Каждый PR катаемо отдельно через `/reliz`.
- **Шрифты** — PT Serif (заголовки + italic-акценты на «жемчужина»/«стоит с 1808 года»), Manrope (sans для body), JetBrains Mono (eyebrows). Все через `next/font/google`.
- **Палитра** — расширяем существующую `--brand-*` системой токенов из handoff'а: `--paper` (#ede3cf), `--paper-deep`, `--ink`, `--forest`, `--forest-deep`, `--ochre`, `--ochre-light`, `--oxblood`, `--muted`, `--rule`. Старые `--brand-forest` / `--brand-amber` / `--brand-clay` остаются как алиасы — чтобы не сломать существующие компоненты.

## Alternatives considered

- **Направление A (Журнал)** — редакционная вёрстка, антиква, full-bleed фото, спокойный воздух. Самый «безопасный» вариант, но не даёт сильной эмоции от первого экрана. Не отвечает аудиту #4 (фото не герой) так же ярко, как B.
- **Направление C (Сторителлинг)** — тёмная сцена, скролл-нарратив (село → храм → люди → ремёсла → отель). Самый «смелый» и эффектный, но требует **много фото с реальной локации** премиум-качества, которых у нас сейчас нет. И тёмный фон плохо ложится на «вятскую» эстетику натурального дерева и глины. Отложен — может быть, для отдельной story-страницы вроде `/o-sele/`.
- **Симбиоз A+B+C** — соблазнительно, но смешение визуальных языков всегда смотрится хуже, чем чистое направление. Лучше выбрать одно и довести до конца.
- **Оставить orbit-карусель на главной + добавить 4-group cards ниже** — пользователь сам отверг: «не люблю когда две модели навигации конкурируют».
- **Сохранить orbit, но переписать только хедер/футер/слаги** — половинчатое решение, аудит #2/#3/#4 не закрываются.
- **Удалить /projects полностью** — теряем admin-инструмент с inline-редактированием (`EditProjectDialog`). Не оправдано.

## Consequences

### Положительные

- Закрываются все 6 пунктов аудита из `handoff-2026-05-23/audit.jsx`.
- Главная становится «лицом», не списком — повышается шанс что турист задержится.
- Иерархия 4 групп даёт **два уровня навигации**: «зачем приехать в Гоньбу» (stay/do/see/shop) → «какой конкретно проект» (внутри группы). Это решает «турист хочет сценарий, а не структуру».
- Mobile-first вёрстка (drawer-меню, `clamp()`-типографика) лечит #6.
- Семантика и SEO улучшаются: Schema.org `TouristAttraction` JSON-LD на главной, осмысленный OG, drawer с подзаголовками для каждого проекта.
- Никаких потерь данных и архитектурных сломов на бэкенде — Payload-коллекции не трогаем.
- Дизайн-токены и шрифты становятся переиспользуемыми для всех будущих страниц (не только главной).

### Отрицательные

- Большой объём работы — 4 PR'а на несколько сессий. До окончания PR3 главная будет жить в «промежуточном» состоянии (либо старая, либо неполная новая) — это надо учесть в стратегии релиза.
- Нужно вручную **сделать маппинг 10 проектов на 4 группы** и проставить флаги `isHero` / `isFeatured` через админку — это ручной шаг на проде после миграции PR2.
- 3 проекта со слагами в навигации (`eco-hotel-booking`, `about-project`, `vyatskiy-sbor`) нужно переименовать **до** релиза PR1 — иначе drawer-меню повторит исходную проблему. Это ручная правка title в админке Payload.
- Цены и длительность мастерских в шаблоне захардкожены (`2ч · 1800₽`). Чтобы вытащить их в админку, нужно либо расширить `Services`, либо завести `Workshops` коллекцию — это потенциальный PR5 (вне основного плана).
- Старые UI-компоненты (`HomeCarouselMenuClient`) живут параллельно с новой главной — пока их обслуживание не стоит ничего, но и моральный долг остаётся.

### Нейтральные

- Главная по высоте станет в 2-3 раза длиннее (раньше — orbit-карусель ≈ 1 viewport; теперь — hero + 4 groups + featured + people + crafts + shop + events + quote = ~6-8 viewport'ов). Это нормально для туристического сайта, но визуальная плотность меняется.
- Появляются 3 новых Google-шрифта (PT Serif, Manrope, JetBrains Mono). Через `next/font/google` self-hosted, без cookie-warning, но всё-таки +ko font weights в bundle.
- Часть существующих SCSS файлов (например, `EditableProjectsGrid` уже использует свои переменные) останется как есть — новая система токенов добавляется поверх через CSS-vars без замены.
