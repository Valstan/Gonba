# 0006. Главная — орбит-карусель; этно-лендинг → `/usadba`

- **Status:** Accepted
- **Date:** 2026-05-30
- **Deciders:** valstan + Claude

## Context

ADR-0004 (2026-05-23) выбрал для главной этно-модерн лендинг (Hero / 4 группы / Глава / Цитата), а интерактивную орбит-карусель проектов увёл на отдельный роут `/orbit`. На практике `/orbit` оказался orphaned (ни одной ссылки), а владелец хочет, чтобы **первое, что видит посетитель** — это живая карта проектов (орбита), а не статичный лендинг.

Параллельно у карусели накопились проблемы: огромный stage (`min-height: 76rem`), дрожащие подписи (кольцо и контр-вращение подписей крутились двумя независимыми CSS-анимациями, дрейфовавшими друг относительно друга), и 12 кружков с дублями/служебными страницами вместо «1 кружок = 1 реальный проект».

## Decision

1. **Главная (`/`) = орбит-карусель** проектов (+ этно-шапка сверху, +`HomeProjectGridMobile` свайп на мобиле). Логика рендера перенесена из `orbit/page.tsx` в `page.tsx`.
2. **Этно-модерн лендинг → `/usadba`** (новый роут). Компоненты `web/src/components/Home/Ethno*` не тронуты, только переехал контейнер-страница.
3. **`/orbit` → `redirect('/')`** (роут больше не нужен, не ломаем закладки).
4. **Курация 1:1** через новое поле `Projects.showInOrbit` (checkbox) — кружок только для `showInOrbit !== false`, центр (`gonba`) исключён. Дубли/инфо выключены (`eco-hotel-booking`, `vyatskaya-lepota-malmyzh`, `about-project`, `village-and-temple`).
5. **Карусель переписана** на единый источник вращения: `requestAnimationFrame` пишет одну CSS-переменную `--orbit-rot`; кольцо вращается на `+rot`, подписи контр-вращаются на `-rot`, читая ту же inherited-переменную → подписи всегда горизонтальны, без дрожания. Компактнее, пауза на hover, `prefers-reduced-motion`.

## Alternatives considered

- **Оставить лендинг на `/`, орбиту на `/orbit`** (статус-кво ADR-0004) — отброшено: владелец хочет орбиту первым экраном; `/orbit` всё равно никто не находил.
- **Своп `/` ↔ `/orbit`** (лендинг на `/orbit`) — отброшено: имя `/orbit` для лендинга нелогично. `/usadba` семантичнее.
- **Удалить этно-лендинг совсем** — отброшено: в него вложено 3 PR'а работы, он ценен как «история усадьбы»; сохраняем на `/usadba`.
- **Курация хардкод-списком слагов / фильтром по `homepageGroup`** — отброшено в пользу явного чекбокса `showInOrbit`: владелец курирует через админку, новые проекты появляются автоматически, дубли выключаются точечно.
- **CSS `@property --orbit-rot` или две CSS-анимации** — отброшено в пользу rAF: гарантированная синхронизация кольца и подписей (ноль дрейфа), явный контроль паузы/reduced-motion.

## Consequences

### Положительные

- Орбита — первый экран; `/orbit` больше не мёртвый роут.
- Подписи не дрожат (единая `--orbit-rot`); карусель компактнее и читаемее.
- Ровно один кружок на реальный проект; нет дублей и пустых переходов.
- Попутно реализован `/projects?group=` фильтр (шапка/футер/группы перестали вести в полный каталог) и исправлены битые слаги в мобильном drawer'е.

### Отрицательные

- Этно-лендинг теперь на шаг дальше (`/usadba`, ссылка в шапке/футере/drawer) — меньше прямой видимости.
- Появилось ещё одно поле курации (`showInOrbit`), которое надо помнить при заведении проектов (дефолт `true` — новый проект сразу в орбите).

### Нейтральные

- `ADR-0004` частично пересмотрен (главная больше не этно-лендинг, но сам этно-лендинг жив на `/usadba`). Статус 0004 → «Superseded in part by 0006».
